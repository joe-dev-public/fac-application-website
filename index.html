Week 09: Build a feature

The design below represents a simple step sequencer, like you might find in a software drum machine, etc.

It won't make any sound (see below for why), and is just intended as a simple UI demonstration and exploration.

There are some options for ways to interact with it, which are based on mouse input.

The purpose of this is to let you work out what combination of mouse inputs feels most fluid and natural for you.

The default options selected below are what the author likes, but you might like something else! :)

You might find that some combinations of actions conflict with one another, or just feel very strange and unintuitive!

It won't work with a touchscreen device yet, as that's outside the scope of this feature for now.

Try clicking a step to toggle it on or off. Steps toggled on are considered "active", and off are considered "inactive".

Why doesn't the sequencer make noise?

I implemented some basic noise-making functionality, based on these examples.

While I understood the concepts and most of the code I was writing, I don't think that I've properly understood an example unless I can change it/rewrite it/write it from scratch to do "whatever I want".

In this case, with the time constraints I gave myself, I couldn't do that. So I didn't think it was fair to call the noise-making code "code I've written", as per the FAC spec for the application website.
